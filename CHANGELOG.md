# Changelog

All notable changes to V4 - Fathom Calls Collector will be documented in this file.

## [0.1.0] - 2026-03-28

### Added
- Initial project scaffold generated by Project Generator
- Complete Next.js 15 application structure with App Router
- Supabase integration with PostgreSQL database
- TypeScript configuration with strict mode
- Tailwind CSS setup for styling

#### Database Schema
- `users` table with authentication and profile management
- `fathom_calls` table for storing call metadata and references
- `call_transcripts` table for storing extracted transcript data
- `action_items` table for voice-to-action item extraction
- `email_notifications` table for tracking notification history
- `api_integrations` table for managing third-party API configurations
- `search_queries` table for search analytics and history
- `processing_jobs` table for background job tracking

#### Route Structure (18 pages)
- Public routes: `/`, `/login`, `/signup`, `/forgot-password`
- Protected dashboard routes: `/dashboard`, `/calls`, `/calls/:id`, `/search`, `/action-items`
- Admin routes: `/sync`, `/notifications`, `/integrations`, `/settings/*`
- API routes: `/api/sync`, `/api/webhooks/fathom`

#### Integration Stubs
- Fathom API integration configuration
- Google Drive API setup for file storage
- Gmail API integration for email notifications
- Twilio SMS integration for text notifications
- HubSpot API configuration (roadmap feature)
- Zapier webhook endpoints
- Make.com webhook endpoints
- n8n webhook endpoints

#### Authentication System
- Supabase Auth integration
- Role-based access control (Owner, Admin, Viewer)
- Row Level Security (RLS) policies for all tables
- Protected route middleware

#### UI Components
- Basic dashboard layout structure
- Authentication forms (login, signup, forgot password)
- Navigation components with role-based visibility
- Loading states and error handling components
- Responsive design with mobile-first approach

#### Documentation Suite
- README.md with complete setup instructions
- CLAUDE.md briefing file for AI development sessions
- TECHNICAL_DEBT.md tracking known shortcuts
- ROADMAP.md with feature planning and hour estimates
- Environment variable configuration template

#### Development Infrastructure
- ESLint and Prettier configuration
- TypeScript strict mode with comprehensive type definitions
- Supabase CLI configuration and migration setup
- Vercel deployment configuration
- Git workflow with conventional commit standards

### Technical Foundation
- Server components by default with selective client components
- Centralized data access layer in `/db` directory
- Business logic separation in `/lib` and `/actions`
- Structured error handling with proper HTTP status codes
- Environment-based configuration management
- Security-first approach with no client-side secrets
